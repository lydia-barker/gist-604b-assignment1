## Repository 1

**Repository Name: Accessibility-Sobi-Hamilton**  
**Link: https://github.com/paezha/Accessibility-Sobi-Hamilton**  

### Purpose
This project uses spatial data to examine the equitability of bike share locations using a method called a floating catchment area approach to increase accuracy of scores.

### Repository Structure
There are three different README files. The README.md file is a scholarly article. There is also a link to the article under the "about" section of the main repository page. There is also a branch called "revisions-3."

### Documentation
The README.md is very helpufl. It includes an abstract of the work, images showing spatial analysis, and outlines the steps that the author took to complete the analysis. 

### Activity
No. There have been 0 commits in the last year and zero pull requests in the last month. 

### Key Observation
One thing this repository does well is that I knew exactly what the project was as soon as I entered the main page. They also added many tags so people interested in this topic could easily find this work.

## Repository 2

**Repository Name: Crash Injury Dashboard**  
**Link: https://github.com/dcvisionzero/Crash-Injury-Dashboard**  

### Purpose
In this repository is a dashboard that allows members of the public to view fatal crash locations and additional information about fatal crashes in DC.

### Repository Structure
There are file types that stand out to me because they are unfamiliar. One of these are the ".parquet" files. After some research, I found out that these file types store binary data in columns rather than rows, creating a smaller and more efficient file type compared to a CSV file, for example.

### Documentation
The README.md is consice and informative. It tells you what data is used (crash outcome, date range, and sources), it provides additional links to the original data, and important data assumptions.

### Activity
Yes, this repository is activley maintained. It was created 9 months ago and the last commit was just a few hours ago. 

### Key Observation
One thing this repository does well is keeping the data clear and consise. Compared to the first repository review, this is a structure that I would be more likely to mimick if I were devleoping a similar project.

## Repository 3

**Repository Name:saferoutes2school**  
**Link: (https://github.com/IndianHillsLawrence/saferoutes2school)**  

### Purpose
This project locates homes around schools, measures the distance to the schools, and creates a tool for finding the best and safest route to get there.

### Repository Structure
The structure is very simple... almost too simple. There is a "rendering" folder with pdfs showing very busy maps that are difficult to read. 

### Documentation
The README is helpful in that it provides each step for running the tool. However, it lacks information on applicability and previous usage.

### Activity
This repository is not actively maintained. It was created 12 years ago and has not had any commits since.

### Key Observation
The license (AGPL) is clear and well layed out.

## Repository 4

**Repository Name:genderbias**  
**Link: https://github.com/slowe/genderbias**  

### Purpose
This is a tool that can be used to calculate gender bias found in letters of recommendation.

### Repository Structure
There is a folder called "examples" that includes a README file and example texts of short letters of recommendation. I am assuming this is to test the tool.

### Documentation
The main README is too short. It does give credit to another similar tool created by another individual (this tool is linked in the README). It allows you to paste the text of a recommendation letter and calculate it for gender bias.

### Activity
It is not actively maintained. It was first created in 2023, and that is when the last commit was.

### Key Observation
I appreciate that it gives examples of letters of recommendation. This allows a user to test the tool without actually needed a letter of recommendation to test it with.
